Learn computing summaries

File Organization

Files on computer are stored in a series of folders or directories. The location of files within your computer is called the path. As we work with files, it is important to have a naming convention. Three guidelines to follow are:

  1. Avoid abbrieviations
  2. Append version numbers and dates
  3. Use descriptive names
There are also established conventions, such as PascalCase, camelCase, snake_case, and so on. It is a good idea to name files iteratively as well. This means using v1, v2, v3 as you make changes instead of relative terms such as "final draft" or "completed". Often times we end up changing these files anyways.

Keyboard Shortcuts

As you start working with web dev, it is important to develop an efficient work flow. Keyboard shortcuts offer a quick way to do tasks that often take multiple clicks. A basic example is CTRL+C on windows to copy highlighted text. This usually takes two additional clicks, a right clicks to bring up the option menu and then selecting the 'copy' option. Combining shortcuts into a routine can make frequent tasks turn into muscle memory. For example, you web-dev edit-save-refresh loop can be done entirely on the keyboard to a faster effect. Keyboard shortcuts are easy to look up if you forget them, but you should memorize all of the shortcuts you use regularly so that they can actually save you time during your work.
